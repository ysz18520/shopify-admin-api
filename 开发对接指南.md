# Shopify 后端开发对接指南

## 一、项目结构

```
shopify网站后端/              ← Node.js + Express + MySQL 后端服务
├── dev-test/                 ← 前端开发测试页面（纯 HTML/CSS/JS）
│   ├── index.html            ← 测试入口
│   └── coollaa/
│       └── booking.html      ← 预约功能测试页
├── src/
│   ├── features/
│   │   ├── booking/          ← C端预约接口
│   │   └── admin/            ← B端管理接口（多店铺权限）
│   ├── middleware/auth.ts    ← JWT鉴权 + siteFilterMiddleware
│   ├── config/booking.ts     ← 多店铺配置映射 SITE_CONFIGS
│   ├── routes/
│   │   ├── coollaa.ts       ← /api/coollaa/*
│   │   └── admin.ts         ← /api/admin/*
│   └── ...
├── prisma/schema.prisma     ← 数据库模型
└── .env                     ← 环境配置

shopify网站后端管理后台/        ← Vue3 + Element Plus 管理后台（独立项目）
├── src/views/
│   ├── Dashboard.vue        ← 概览统计（支持超管店铺筛选）
│   ├── BookingList.vue      ← 预约列表（支持店铺筛选）
│   ├── AvailabilityConfig.vue ← 可用性配置（超管需选店铺）
│   ├── PreviewTest.vue      ← 页面预览（支持店铺筛选）
│   └── ApiInfo.vue          ← 接口信息
└── ...
```

## 二、服务启动方式

### 1. 启动数据库（Docker）

```bash
docker run -d \
  --name mysql-shopify \
  -e MYSQL_ROOT_PASSWORD=password \
  -e MYSQL_DATABASE=shopify_backend \
  -p 3306:3306 \
  mysql:8
```

> 数据库配置在 `shopify网站后端/.env`：
> ```
> DATABASE_URL=mysql://root:password@localhost:3306/shopify_backend
> ```

### 2. 启动后端服务

```bash
cd shopify网站后端
npm install
npx prisma migrate dev --name init    # 首次执行
npx prisma generate
npm run dev                           # http://localhost:3000
```

### 3. 启动管理后台

```bash
cd shopify网站后端管理后台
npm install
npm run dev                           # http://localhost:5173
```

| 服务 | 地址 | 账号/密码 |
|------|------|----------|
| 后端 API | `http://localhost:3000` | - |
| 管理后台 | `http://localhost:5173` | admin / admin（超管）或 coollaa / coollaa（店铺） |

## 三、前端测试页面

**`dev-test/coollaa/booking.html`** 是纯 HTML/CSS/JS 编写的预约功能页面：

- **目的**：在 Shopify 上线前测试功能完备性，调通 API 后直接搬运到 Shopify
- **技术栈**：零依赖，原生 JavaScript + Fetch API
- **修改点**：文件第 3 行 `API_BASE` 改为你的域名即可上线

```javascript
const API_BASE = 'http://localhost:3000/api';   // 开发
// const API_BASE = 'https://api.coollaa.com/api'; // 上线
```

**浏览器直接打开**：`dev-test/index.html` → 点击 Coollaa → 预约功能

## 四、C端 API 接口（给 Shopify 前端用）

Base URL: `http://localhost:3000/api`

### 4.1 获取商家配置

```
GET /coollaa/booking/config
```

**响应**：
```json
{
  "timeZone": "Asia/Shanghai",
  "timeZoneName": "UTC +08:00 中国、中国香港、新加坡",
  "durationOptions": [30, 60],
  "intervalMinutes": 15,
  "meetingOptions": ["WhatsApp Call", "In-Person Appointment"]
}
```

### 4.2 查询可用时段

```
GET /coollaa/booking/slots?date=2026-05-15&duration=30&timezone=Asia/Shanghai
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `date` | 是 | 日期 `yyyy-MM-dd`（商家时区） |
| `duration` | 是 | 时长：30 或 60 |
| `timezone` | 是 | 用户时区，如 `Asia/Shanghai` |

**响应**：
```json
{
  "date": "2026-05-15",
  "duration": 30,
  "slots": [
    { "utc": "2026-05-15T01:00:00.000Z", "localTime": "09:00", "localDate": "2026-05-15" },
    { "utc": "2026-05-15T01:15:00.000Z", "localTime": "09:15", "localDate": "2026-05-15" }
  ]
}
```

> 已被预约、午休（12:00-14:00）、节假日、周末、过去时段均不返回。

### 4.3 创建预约

```
POST /coollaa/booking
Content-Type: application/json
```

**请求体**：
```json
{
  "date": "2026-05-15",
  "startTime": "09:00",
  "duration": 30,
  "userTimezone": "Asia/Shanghai",
  "lastName": "张",
  "firstName": "三",
  "email": "zhangsan@example.com",
  "company": "ABC公司",
  "phone": "13800138000",
  "meetingType": "WhatsApp Call"
}
```

**必填字段**：`date`, `startTime`, `duration`, `userTimezone`, `lastName`, `company`, `phone`, `meetingType`

**响应**：
```json
{
  "id": "cm8xxxxx",
  "site": "coollaa",
  "startTime": "2026-05-15T01:00:00.000Z",
  "endTime": "2026-05-15T01:30:00.000Z",
  "duration": 30,
  "lastName": "张",
  "firstName": "三",
  "email": "zhangsan@example.com",
  "company": "ABC公司",
  "phone": "13800138000",
  "meetingType": "WhatsApp Call",
  "userTimezone": "Asia/Shanghai",
  "status": "confirmed",
  "createdAt": "2026-05-13T08:30:00.000Z",
  "updatedAt": "2026-05-13T08:30:00.000Z"
}
```

**错误码**：
| 状态码 | 说明 |
|--------|------|
| 400 | 参数缺失 / 时长错误 / 过期时段 |
| 409 | 该时段已被预约 |
| 500 | 服务器内部错误 |

## 五、管理后台 API 接口

Base URL: `http://localhost:3000/api/admin`

所有接口（除登录外）需要在 Header 中携带：`Authorization: Bearer {token}`

### 5.1 管理员登录

```
POST /admin/login
Content-Type: application/json

{ "username": "admin", "password": "admin" }
```

**账号体系**：
| 账号 | 密码 | 角色 | 说明 |
|------|------|------|------|
| `admin` | `admin` | `super` | 超管，可查看所有店铺 |
| `coollaa` | `coollaa` | `site` | 店铺管理员，只能看 coollaa 数据 |
| `longshade` | `longshade` | `site` | 店铺管理员，只能看 longshade 数据 |

**响应**：
```json
{
  "token": "xxx",
  "username": "admin",
  "role": "super",
  "site": "all"
}
```

### 5.2 预约列表

```
GET /admin/bookings?page=1&pageSize=10&status=confirmed&site=coollaa
```

| 参数 | 说明 |
|------|------|
| `page` | 页码，默认 1 |
| `pageSize` | 每页条数，默认 10 |
| `status` | 可选，按状态筛选 |
| `site` | **超管可选**，按店铺筛选（不传则返回全部） |

> 店铺管理员自动只返回自己店铺的数据，无需传 site。

### 5.3 取消预约

```
PUT /admin/bookings/:id/cancel
```

> 店铺管理员只能取消自己店铺的预约。

### 5.4 统计数据

```
GET /admin/stats?site=coollaa
```

| 参数 | 说明 |
|------|------|
| `site` | **超管可选**，按店铺统计（不传则返回全部店铺汇总） |

**响应**：`{ "today": 0, "week": 0, "month": 0, "pending": 0, "total": 0 }`

### 5.5 可用性配置

```
GET /admin/availability?site=coollaa
PUT /admin/availability?site=coollaa
Body: { "availability": [...], "breaks": [...] }
```

> `site` 为必填参数（店铺角色由拦截器自动附加）。

### 5.6 特殊日期（节假日）

```
GET /admin/holidays?site=coollaa
POST /admin/holidays?site=coollaa       Body: { "date": "2026-01-01", "reason": "元旦" }
DELETE /admin/holidays/:id?site=coollaa
```

> 所有操作均按店铺隔离。

## 六、数据库模型

| 表名 | 用途 |
|------|------|
| `Booking` | 预约记录（含 `site`, `userTimezone` 字段） |
| `AvailabilityConfig` | 工作日时间配置（周一到周日，含 `site`） |
| `BreakTime` | 午休/休息时间配置（含 `site`） |
| `Holiday` | 特殊节假日（不可预约日期，含 `site`） |

## 七、前端对接流程（Shopify）

1. **初始化**：`GET /booking/config` → 渲染时长选项、时区、会议方式
2. **选日期**：展示日历，用户点击日期 → `GET /booking/slots?date=...` → 渲染可用时段列表
3. **选时段**：用户点击时段 → 记录 `date` + `startTime`（商家时区 `HH:mm`）
4. **填信息**：展示表单（姓氏*、名字、邮箱、公司*、电话*、会议方式*）
5. **提交**：`POST /booking` → 成功展示确认页，失败（409）提示重新选择

> 参考实现：`dev-test/coollaa/booking.html`（原生 JS，可直接搬运到 Shopify）
