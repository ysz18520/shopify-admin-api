## 后端变更

### 1. JWT 鉴权扩展 (`src/middleware/auth.ts`)
- Payload 结构: `{ username, role: 'super' | 'site', site: string }`
- 新增 `siteFilterMiddleware`: 店铺角色强制注入 `req.query.site`，超管可自由筛选

### 2. 多账号登录 (`src/features/admin/service.ts`)
- 移除 bcrypt 硬编码密码方案
- `admin/admin` → role='super', site='all'
- `{site}/{site}` → role='site', site=店铺名
- 当前支持店铺: `coollaa`, `longshade`

### 3. Admin API 数据隔离 (`src/features/admin/controller.ts` + `routes/admin.ts`)
- 所有 admin 路由增加 `siteFilterMiddleware`
- `getBookings` / `getStats`: 超管不传 site 返回全部，传 site 返回单个
- `getAvailability` / `saveAvailability` / `getHolidays` / `addHoliday` / `removeHoliday`: 必须传 site
- `cancelBooking`: 增加权限校验（只能取消自己 site 的预约）

### 4. 多店铺配置 (`src/config/booking.ts`)
- `COOLLAA_BOOKING` → `SITE_CONFIGS` 映射表
- `getSiteConfig(site)` 动态获取配置
- `getAllSites()` 返回所有店铺名

### 5. Prisma Schema
- Booking 模型新增 `userTimezone String @default("Asia/Shanghai")`

## 前端变更 (管理后台)

### 1. Auth Store (`src/stores/auth.ts`)
- 新增 `role`, `site`, `viewSite` ref
- 新增 `isSuper` computed
- 新增 `setViewSite()` 方法（超管切换查看店铺）
- 登出时清除所有 localStorage 字段

### 2. Request 拦截器 (`src/api/request.ts`)
- 店铺角色自动在所有请求 query 中附加 `site` 参数
- 超管请求不自动附加 site（由页面控制）

### 3. Admin API (`src/api/admin.ts`)
- `AdminLoginResponse` 新增 `role` 和 `site`
- 所有函数增加可选 `site` 参数（供超管筛选使用）

### 4. 页面适配

| 页面 | 变更内容 |
|------|----------|
| Layout.vue | 动态标题（超管显示"管理后台(超管)"，店铺显示"管理后台-{site}"），右上角角色标签 |
| Dashboard.vue | 超管显示店铺筛选器，支持查看全部/单个店铺统计 |
| BookingList.vue | 超管显示店铺列 + 店铺筛选器 |
| AvailabilityConfig.vue | 超管需先选店铺才能查看/编辑配置 |
| PreviewTest.vue | 超管可按店铺筛选测试页面 |
| ApiInfo.vue | 显示当前账号角色和所属店铺/查看范围 |
