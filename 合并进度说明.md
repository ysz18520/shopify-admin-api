## 一、已完成的工作

### 1. 后端 API 合并 ✅

已将 `voting-system` 项目的功能合并到 `shopify-admin-api`：

| 模块 | 文件 | 说明 |
|------|------|------|
| designs | `src/features/designs/` | 设计作品管理（CRUD + 投票） |
| contacts | `src/features/contacts/` | 联系人信息收集 |
| routes | `src/routes/designs.ts` | 设计相关路由 |
| routes | `src/routes/contacts.ts` | 联系人相关路由 |
| utils | `src/utils/helpers.ts` | 工具函数 |
| utils | `src/utils/response.ts` | 响应格式 |

**API 接口**：
- `GET /api/designs` - 获取设计列表
- `POST /api/designs` - 创建设计
- `PATCH /api/designs/:id` - 更新设计
- `DELETE /api/designs/:id` - 删除设计
- `POST /api/designs/:id/vote` - 投票
- `POST /api/contact` - 提交联系人

### 2. 管理后台页面 ✅

已创建 `DesignList.vue` 设计管理页面：
- 支持搜索、筛选、分页
- 支持添加、编辑、删除设计
- 店铺用户只能管理自己店铺的数据

### 3. 店铺账号系统 ✅

- 添加店铺自动创建同名账号（店铺名/店铺名）
- 登录从数据库验证
- 超管可以为店铺开启/关闭功能

### 4. 功能开关 ✅

在超管的店铺管理页面，可以为每个店铺开启/关闭功能：

| 功能 | 开关字段 | 说明 |
|------|----------|------|
| 预约功能 | `isBookingEnabled` | 开启后显示预约相关页面 |
| 投票功能 | `isVotingEnabled` | 开启后显示投票相关页面 |

### 5. 权限控制 ✅

| 用户类型 | 可见菜单 |
|----------|----------|
| 超管 | 全部菜单 |
| 店铺用户 | 店铺信息、页面预览 + 已开启的功能页面 |
